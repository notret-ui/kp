# Бриф для Claude design — прототип КП «Печь.ру»

Задача: собрать **статичный HTML/CSS-прототип** (мокапы, без бэкенда и без данных)
всех слайдов КП строго по дизайн-системе. Результат складывать в папку `prototype/`.

## Обязательно к прочтению перед стартом
- `docs/design-system.md` — токены, компоненты, типы слайдов. **Это источник правды.**
- `reference/КП 200825 ДАО.pdf` — оригинальный Figma-шаблон, повторять визуал по нему.
- `reference/page1..4.png`, `reference/figma_overview.png` — скриншоты слайдов.

## Что сверстать (каждый слайд — отдельный экран)
Десктоп (ландшафт 16:9) **и** мобайл (портрет) для каждого:

1. **Обложка** — оффер + 6 преимуществ + место под имя клиента и дату.
2. **О компании** «с 2009 г.» + фото.
3. **Карточка товара** — плашка менеджера сверху; фото слева; название + таблица
   характеристик (2 колонки) справа; крупная коралловая цена + зачёркнутая старая; описание.
4. **Итоги** *(нового нет в Figma — спроектировать в том же стиле)*: таблица позиций
   (миниатюра, название, кол-во, цена, сумма), отдельные строки доставка / монтаж / услуги,
   скидка на КП, строка ИТОГО.
5. **Галерея** «Фото наших довольных клиентов» (сетка 2×3).
6. **Товары в этой категории** — ряд мини-карточек.
7. **Контакты** — 4 карточки (многоканальный телефон, мессенджеры, email, время работы).

На всех слайдах — коралловый сайдбар справа с лого «Печь.ру», водяным знаком и ссылкой
«Подробнее на Печь.ру».

## Правила
- Только статика: HTML + CSS (можно один `tokens.css` с переменными из дизайн-системы).
- Данные — заглушки/плейсхолдеры (имена, цены, фото-боксы). Реальные данные подключит Claude Code.
- Вёрстка должна легко превращаться в Jinja2-шаблоны (понятные блоки/классы под компоненты).
- Сначала показать мокапы на ревью → после апрува передаём обратно в Claude Code.

## Чего НЕ делать
- Не писать бэкенд, не тянуть фид/БД, не подключать Playwright — это фаза реализации.
